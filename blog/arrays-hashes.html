What are arrays and hashes and when would you use them?
6
Nov

Arrays and hashes are index collections used to store and access collections. Both have their advantages and disadvantages in how they are stored and accessed, so choosing the right one is important. First we'll talk a little bit about what each one is.

Arrays are sequentially ordered lists. It can store all sorts of objects, even other arrays. They look like this [ 0 , 1, 2, 3, 4, 5]. The 0 is in the 0th position and the 4 is in the 4th position. Notice though that we start at 0, so a position of 5 stores 6 values.So these can actually store any kinds of objects like strings, poems, number, methods, or, like we said before, other arrays.

Hashes are another index collection that store information in key-value pairs. Again these can store any kind of objects, even other arrays or hashes! So these key-value pairs store information that can be recalled from the key. For example if I'm doing a dossier on a person, I can call the keys by Age, Last Name, Occupation and load this information in constant time.

Here's something I read that really helped me with these concepts. You can think of an Array like it's a grocery list. You have your eggs, milk and cereal. They have no special order, they are just stored in this list as top to bottom. For hashes I like to think of them like a set of flash cards that you used to study from. Like I can have something that goes key-value like "Hello"-"A standard greeting". This way hashes act as a definition.

Hashes storing hashes can be great for storing user information. We can have a hash called "users" with names like "calvin" and "joe." But then within the "calvin" key, you can store another hash with the keys of "age" and "email".

The advantage of each is how you want to use your data and access it. Arrays can have iterations that run through each and every value to access it. The utility of this is incredibly but in a way it's like going through a telephone book from A-Z to find something that starts with K. But if you want to find every single one that is longer than 7 letters long and array is your ideal structure. Whereas with Hashes we can do a direct access that takes a constant amount of time for each value. This would make looking for a single value much easier and is better for lookup.

In the end, look for how you want you data to be processed and the algorithms you want to use. That will determine whether you want to use hashes or arrays.
